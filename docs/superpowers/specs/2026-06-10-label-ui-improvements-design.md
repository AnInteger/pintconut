# 板子标注 UI 改进设计

> 日期：2026-06-10
> 范围：`src/label_ui.py` + `src/label_service.py`

## 背景

用户在实际使用板子标注流程时遇到 4 个问题：

1. **区域编号不明显** — 图片上的编号难以与按钮对应
2. **豆子区域干扰** — 板子上有豆子时，FastSAM 把豆子区域也分割为候选
3. **完成后无引导** — 全部标注完后没有明显的"下一步"引导
4. **缺少结果展示** — 标注后无法在图片上看到最终标注效果

## 改动清单

### 改动 1：候选按钮增加颜色标识

**文件**：`src/label_ui.py` — `_load_review_image()`

**现状**：按钮文字为 `#0 面积 45.2%`，与图片上区域的颜色无对应关系。

**改为**：按钮文字中嵌入对应颜色的 emoji 色块。定义一组与 `COLORS` 顺序对应的 emoji：

```python
COLOR_EMOJIS = ["🟢", "🔴", "🔵", "🟡", "🟦", "🟣", "🟩", "🔶", "🟠", "🟤"]
```

按钮文字改为：`🟢 #0 面积 45.2%`

用户对照逻辑：看图片上区域的颜色 → 找同颜色 emoji 的按钮 → 点击确认。

**不改动**：图片上的渲染逻辑（`draw_candidates`）保持不变，已有彩色半透明覆盖 + 白色编号，可读性足够。

---

### 改动 2：标注完成后显示"导出数据集"引导按钮

**文件**：`src/label_ui.py` — `_handle_select_candidate()` / `_handle_skip_photo()`

**现状**：全部标注完成后，图片区域清空，显示文字 `"🎉 全部处理完成！请点击上方「④ 导出数据集」标签继续"`，但没有可操作的按钮。

**改为**：

- 全部完成后，在标注确认区域显示一个**醒目的"📦 导出数据集"按钮**
- 该按钮点击后自动跳转到 Tab 4（导出数据集）并触发导出
- 实现方式：新增一个 `gr.Button`，仅在全部完成时变为可见

---

### 改动 3：标注后展示结果图

**文件**：`src/label_service.py` — 新增 `draw_result()` 函数

**功能**：在用户选择候选区域后，在原图上绘制标注结果：

- 绿色轮廓线标出选中区域的 mask 边界
- 轮廓上方标注 `"✅ 板子"` 文字
- 无半透明覆盖，只画轮廓线，保持原图清晰

**文件**：`src/label_ui.py` — `_handle_select_candidate()`

**改为**：选中区域后，不立即跳到下一张，而是：

1. 在当前图片上展示标注结果（调用 `draw_result()`）
2. 显示 2 个按钮：
   - `"✅ 确认，下一张"` — 保存标注并自动跳到下一张
   - `"↩️ 重新选择"` — 回到候选区域视图重新选择

用户必须点"确认，下一张"才保存并跳转。不自动跳转——让用户有机会在结果图上确认标注是否正确。

**注意**：跳过的图片不展示结果，直接下一张。

---

## 不改动的内容

- **分割逻辑**：`segment_image()` 不改，豆子区域干扰是 FastSAM 本身的特性，通过标注时人工判断解决
- **合并区域**：不做多选合并，保持单选
- **自动跳转**：保持现有的自动下一张行为
- **数据格式**：YOLO 分割格式不变
