# 标注 UI 重构设计（V2）

日期：2026-06-09

## 背景

V1 使用 Gradio Tabs 实现 4 步标注流程，存在以下问题：

1. **Tab 分步不自然** — 上传完成后没有引导去下一步，用户不知道该点哪
2. **标注方式不直观** — 用 Image index / Candidate index 数字输入框，用户不理解含义
3. **缺乏整体说明** — 不清楚为什么要标注、标注数据用来做什么
4. **进度追踪缺失** — 不知道哪些标了哪些没标

## 目标

严格向导式流程，类似软件安装的 Next → Next → Finish：
- 做完一步才能进下一步
- 每步有明确引导
- 标注操作直观（点击卡片选候选区域）
- 进度可视化

## 设计

### 整体布局

```
┌──────────────────────────────────────────────┐
│  ① 上传照片 ━━ ② 自动分割 ━━ ③ 标注确认 ━━ ④ 导出  │  ← 进度条
├──────────────────────────────────────────────┤
│                                              │
│  [当前步骤内容区域]                              │
│                                              │
│                          [下一步 →] / [← 上一步]│
└──────────────────────────────────────────────┘
```

- 顶部进度条始终可见，已完成步骤 ✅，当前步骤高亮，未解锁步骤灰色
- 中间区域只显示当前步骤内容
- 底部导航按钮

### 步骤 ① 上传照片

**可见条件**：启动时默认显示

**内容**：
- 说明文字：「上传拼板照片，用于训练拼板检测模型」
- 拖拽上传区域（`gr.File(file_count="multiple")`）
- 上传后显示缩略图预览（`gr.Gallery`）
- 状态文字：「已上传 X 张照片」

**导航**：
- 「下一步 →」按钮，上传 ≥1 张后可用

### 步骤 ② 自动分割

**可见条件**：步骤 ① 完成（至少 1 张照片）

**内容**：
- 说明文字：「FastSAM 将自动识别每张照片中的候选区域」
- 「开始分割」按钮
- 分割进度条（`gr.Progress`）
- 分割完成后显示结果缩略图（每张图叠加候选区域轮廓和编号）

**导航**：
- 「← 上一步」返回步骤 ①
- 「下一步 →」分割完成后可用

### 步骤 ③ 标注确认

**可见条件**：步骤 ② 完成

**核心目的说明**：
> FastSAM 自动识别了多个区域，你需要选出**哪个区域是拼板**。
> 选中的结果将作为训练数据，教会 AI 自动识别拼板位置。

**布局**：

```
┌─────────────────────────────────────────────┐
│  左侧：缩略图列表          │  右侧：大图 + 操作  │
│  ┌───┐ ┌───┐ ┌───┐       │                  │
│  │✅ │ │🟡 │ │🟡 │       │   [大图 + 候选区域] │
│  └───┘ └───┘ └───┘       │   ①绿色 ②红色 ③蓝色│
│  img1  img2  img3         │                  │
│                           │  候选区域卡片：     │
│  进度：1/3 已标注           │  [① 面积 38.7%]  │
│                           │  [② 面积 25.1%]  │
│                           │  [③ 面积 15.3%]  │
│                           │                  │
│                           │  [跳过] [确认选①]  │
└─────────────────────────────────────────────┘
```

- 左侧缩略图列表：已标注 ✅，未标注 🟡，跳过 ⏭
- 右侧大图：当前照片 + 候选区域叠加（编号 ①②③，不同颜色）
- 候选区域卡片：每个候选显示编号和面积占比，**点击卡片即选中该候选为拼板**
- 选中后自动跳到下一张未标注照片
- 「跳过」按钮（该照片没有拼板或不满意分割结果）
- 底部显示进度：「2/5 已标注，1 跳过」

**导航**：
- 「← 上一步」返回步骤 ②
- 「下一步 →」所有照片处理完后可用（全部标注或跳过）

### 步骤 ④ 导出数据集

**可见条件**：步骤 ③ 完成

**内容**：
- 统计信息：
  - 已标注 X 张
  - 跳过 Y 张
  - 总照片 Z 张
- 「导出数据集」按钮
- 导出完成后显示：
  - 数据集路径
  - 训练命令：`python training/train.py --data training/dataset/data.yaml`

**导航**：
- 「← 上一步」返回步骤 ③
- 「完成 ✅」按钮

## 技术实现

### Gradio 可见性控制

使用 `gr.Column(visible=...)` 控制每个步骤的显示/隐藏：

```python
with gr.Column(visible=True) as step1:
    # 步骤 ① 内容
    ...
with gr.Column(visible=False) as step2:
    # 步骤 ② 内容
    ...
```

每个「下一步」按钮的回调函数返回：
1. 当前步骤 Column → `visible=False`
2. 下一步骤 Column → `visible=True`
3. 进度条状态更新

### 进度条

用 `gr.Markdown` 实现，根据当前步骤动态更新文本和 emoji：

```python
progress_display = gr.Markdown("① **上传照片** ━━ ② 自动分割 ━━ ③ 标注确认 ━━ ④ 导出")
```

### 步骤 ③ 候选区域交互

由于 Gradio 不支持直接点击图片区域，采用**候选卡片按钮**方案：
- 每个候选区域显示为一个 `gr.Button`，文字为「① 面积 38.7%」
- 点击按钮触发选中逻辑
- 大图上用 OpenCV 绘制编号和颜色对应
