# Pintconut - 拼豆差异检测器设计文档

> 日期: 2026-06-08
> 状态: 已确认

## 1. 项目概述

Pintconut 是一个命令行工具，用于比较拼豆作品照片与拼豆图纸，自动检测并标注出拼错的位置。用户拍摄拼板上的拼豆照片，提供对应的图纸图片，工具输出标注了差异位置的图片。

### 核心场景

- 用户正在拼豆，拍了张照片，想快速知道哪里拼错了
- 输入：拼板照片（手机拍摄）+ 图纸图片（带颜色的网格图）+ 拼板尺寸
- 输出：标注了错误位置的图片（红框标记）

## 2. 技术方案

### 三阶段流水线

```
用户照片 ──→ ① 定位拼板 ──→ ② 提取网格 ──→ ③ 颜色对比 ──→ 输出标注图
用户图纸 ──────────────────────────────────↗
```

| 阶段 | 做什么 | 技术 | 需要模型？ |
|------|--------|------|-----------|
| ① 定位拼板 | 从照片中找到拼板区域并裁剪 | YOLOv8-seg（迁移学习） | 是 |
| ② 提取网格 | 透视校正，将拼板区域切割为规则网格 | OpenCV | 否 |
| ③ 颜色对比 | 逐格判断有豆/无豆及颜色，与图纸对比 | NumPy（LAB 色彩空间） | 否 |

**只有阶段①需要模型，阶段②③用传统算法。**

### 为什么需要模型？

- 拼板在照片中的位置、角度、光照、背景都不固定
- 拼板外观虽然统一（白色边框 + 网格 + 彩色小点），但纯算法难以从复杂背景中可靠定位
- 通过 YOLOv8-seg 迁移学习，只需少量数据即可训练出定位模型

### 为什么只用模型做定位？

- 网格提取是几何问题：拼板钉子是规则排列，透视校正后直接等分即可
- 颜色对比是数学问题：在 LAB 色彩空间做最近邻匹配，比模型更快更准确
- 最小化模型依赖，降低开发和调试复杂度

## 3. 模型训练流程

### 3.1 收集照片

- **数量**: 起步 50 张，效果不够再补充（目标 100-150 张）
- **多样性**: 不同角度、不同光照（日光/灯光）、不同背景（桌面/布面/手中）
- **拼板状态**: 有豆的、空的、部分填充的都要拍
- **设备**: 手机即可

提供 `training/collect_helper.py` 脚本，批量重命名照片为序号编号。

### 3.2 半自动标注

使用 FastSAM 进行半自动标注，大幅减少手工工作量：

**流程：**
1. FastSAM 对每张照片进行分割，生成候选区域
2. 筛选出面积合理（15%-80%）且接近矩形的候选区域
3. 在图片上标注所有候选区域的编号和轮廓
4. 用户查看图片，输入正确的区域编号
5. 脚本自动将选中区域转为 YOLO segmentation 格式标注

**用户工作量**: 每张图只需输入一个数字，50 张图约几分钟。

**工具**: `training/semi_auto_label.py`

### 3.3 训练

基于 YOLOv8n-seg 预训练权重做迁移学习（fine-tuning）：

```bash
python training/train.py --data training/dataset/data.yaml
```

- 模型: `yolov8n-seg.pt`（最小版本，速度快，适合单类别检测）
- Epochs: 100
- 图片尺寸: 640
- 单类别: `beadboard`

**硬件要求：**

| 环境 | 预估时间 | 说明 |
|------|---------|------|
| GPU（RTX 3060+） | 30-60 分钟 | 推荐 |
| Google Colab 免费 GPU | 30-60 分钟 | 无本地 GPU 的选择 |
| CPU only | 3-6 小时 | 可行但较慢 |

**工具**: `training/train.py`

### 3.4 验证

```bash
python training/validate.py --model training/runs/beadboard-v1/weights/best.pt --image test.jpg
```

关键指标：
- mAP50 > 0.9（定位准确度）
- mAP50-95 > 0.7（严格准确度）

验证失败时，针对性补充照片重新训练。

**工具**: `training/validate.py`

### 3.5 迭代

| 问题 | 解决方式 |
|------|---------|
| 特定角度识别不好 | 补充该角度照片 |
| 某种光照下不行 | 补充该光照照片 |
| 整体效果一般 | 增加 epochs 或换 yolov8s-seg |

## 4. 推理流程详解

### 4.1 阶段①：定位拼板

```
原始照片 → YOLOv8-seg 推理 → 获取拼板 mask → 从 mask 提取四角点
```

- 加载训练好的模型权重
- 对输入照片做推理，获取拼板区域的分割 mask
- 从 mask 轮廓提取四个角点（凸包 → 角点检测）

**模块**: `src/detect.py`

### 4.2 阶段②：提取网格

```
四角点 → 透视校正为矩形 → 按拼板尺寸等分为网格 → 每格中心区域采样颜色
```

- 使用 `cv2.getPerspectiveTransform` 将拼板校正为标准矩形
- 根据用户指定的拼板尺寸（如 29×29）等分为网格
- 在每个格子的中心区域采样颜色（避开边缘和间隙）
- 判断每格是"有豆"还是"空"：与拼板底色（通常白色/透明）的色差超过阈值即为有豆

**模块**: `src/grid.py`

### 4.3 阶段③：颜色对比与差异标注

```
照片网格矩阵 vs 图纸网格矩阵 → 逐格比较 → 标注差异位置 → 输出标注图
```

- 将照片中每格采样颜色映射到 221 色盘（LAB 色彩空间最近邻匹配）
- 解析图纸图片，提取每格目标颜色
- 逐格比较，记录差异（颜色不同、应有豆但空了、应空但有豆）
- 在照片上用红框标注差异位置，输出标注图

**模块**: `src/color.py`、`src/blueprint.py`、`src/compare.py`

## 5. 图纸解析

- 图纸是带颜色的网格图片（PNG/JPG），可能包含作者标识
- 需要检测网格线，提取每格的颜色值
- 将颜色映射到 221 色盘
- 需要处理图纸中可能存在的非网格区域（作者标识、边距等）

**模块**: `src/blueprint.py`

## 6. 颜色体系

- 固定 221 种颜色
- 颜色匹配在 LAB 色彩空间进行（比 RGB 更接近人眼感知）
- 每格颜色映射到 221 色盘中距离最近的颜色
- 有豆/无豆判断：与拼板底色的色差阈值

**数据文件**: `data/colors.json`（221 色盘定义，含 RGB 和 LAB 值）

## 7. 拼板尺寸

- 内置常见拼板尺寸规格
- 用户通过命令行参数选择

**数据文件**: `data/board_sizes.json`

## 8. 命令行接口

```bash
# 完整用法
python -m src.cli \
  --photo photo.jpg \         # 拼豆照片
  --blueprint blueprint.png \  # 图纸图片
  --board-size 29x29 \         # 拼板尺寸
  --model models/beadboard-best.pt \  # 模型权重
  --output result.jpg          # 输出路径（可选，默认 annotated_result.jpg）
```

**模块**: `src/cli.py`

## 10. 依赖

- Python 3.10+
- ultralytics（YOLOv8-seg + FastSAM）
- opencv-python（透视校正、网格提取、标注绘制）
- numpy（颜色匹配、矩阵运算）
- Pillow（图片读写）

## 11. 已验证的技术可行性

- FastSAM 已在本地测试环境中成功运行
- 使用 CPU-only PyTorch（无需 GPU 即可推理）
- 自动标注测试结果：FastSAM 能分割出候选区域，但需要人工点选确认（半自动方案）
- 测试图片：3 张不同场景（白色桌面、木质桌面、布面背景）
